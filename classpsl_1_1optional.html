    template<typename T>
    psl::optional class

    wrapper type to optionally hold an object with continuation (then) functionality.

    Template parameters
    T type to contain.

    Contents

    • Public types
    • Constructors, destructors, conversion operators
    • Public functions

    Unlike psl::expected, when optionals are passed in continuation contexts, them being missing is not an error state, but rather a possible expected state. This means that optionals when the continuation doesn't expect an optional, it will have to depend on the fallback value (if provided), or the default value (if possible). Unlike psl::expected, optionals can be re-assigned after construction.

    template<typename T> template<typename FN, typename... Args>
    auto then(FN&& fn, Args && ... args) constexpr

    Calls the given functional object and returns an optional.

    This version will be called when the first argument is an optional and the function does not return an optional. It will wrap the result into an optional.
