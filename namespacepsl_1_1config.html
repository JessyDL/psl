Typedef documentation

using psl::config::default_abstract_memory_resource_t = psl::abstract_memory_resource<psl::traits::shareable_t<true>, psl::traits::basic_allocation>

Default abstract memory resource to use.

Can be used to override the base abstract resources the built-in memory_resource's use. Do be cautious of this setting.

using psl::config::default_memory_resource_t = psl::new_resource

Default memory resource that will be used by containers, or anything needed allocations.

Can be used to override the memory_resource used by most collection type classes. Be mindful that the new resource does need to adhere to the traits.

using psl::config::default_allocator_t = psl::allocator<psl::traits::shareable_t<true>, psl::traits::basic_allocation>

Default allocator to use by any API that exposes allocator usage.

Note that even though yuo can swap allocators out, their traits still have to work for every collection out there.

Variable documentation

static exceptions_handling psl::config::errors_strategy_v constexpr

How to handle errors detected at runtime.

This setting allows the behaviour of runtime errors to be set. Note that even with full exceptions settings enabled, some errors will only present themselves as asserts. It is also possible to have the exceptions as asserts, while the traditional asserts are disabled. This means asserts will still happen, but only in exceptional scenarios.

static exceptions_handling psl::config::implementation_errors_strategy_v constexpr

tracks implementation errors on the part of the programmer

unlike the previous exception handling, which is meant to protect against runtime errors, this variation is meant to check for errors in contract implementation. Most commonly this is used when inheriting from polymorphic base types
